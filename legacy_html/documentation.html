Setup Guide

Get UnityAI running inside your IDE or AI client with the real local proxy URL, UnityAI model IDs, and Anthropic-compatible endpoints.

Drop-in Compatible Per-Key Budgets Proxy Auth UnityAI Models

Prerequisites

Node.js 18 or newer Required for running the UnityAI CLI wizard with npx.
UnityAI API key Create or receive a customer API key from your workspace.
Supported coding tool Claude Code, OpenAI Codex, OpenCode, Cline, Kilo Code, or any OpenAI/Anthropic compatible client.

Quick Setup

Run the UnityAI setup wizard, paste your API key, then choose the coding tool you want to configure.

Main Command 
npx -y @unityai/cli@latest
Connection Values 
API Origin: https://api.unityai.cloud
Anthropic-compatible URL: https://api.unityai.cloud/v1
OpenAI-compatible URL: https://api.unityai.cloud/api/v1
API Key: YOUR_API_KEY
Primary model: unityai-plus
Fast model: unityai-plus

Interactive Setup Flow

The wizard keeps the API key out of the shell history, detects installed tools, and writes the right config for the selected tool.

What the user sees

1
UnityAI banner appears in the terminal.
2
User pastes the UnityAI API key when prompted.
3
Wizard validates the key and lists available coding tools.
4
User enters the tool number, then UnityAI writes the configuration files.
5
User opens or restarts the selected coding tool.

Claude Code CLI

Recommended: run the UnityAI wizard and select Claude Code from the tool list. The wizard writes ~/.claude/settings.json for you.

~/.claude/settings.json 
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "YOUR_API_KEY",
    "ANTHROPIC_BASE_URL": "https://api.unityai.cloud/v1",
    "ANTHROPIC_MODEL": "unityai-plus",
    "ANTHROPIC_SMALL_FAST_MODEL": "unityai-plus",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "claude-opus-4-8",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "claude-sonnet-4-6",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "unityai-plus",
    "CLAUDE_CODE_SUBAGENT_MODEL": "unityai-plus",
    "CLAUDE_CODE_MAX_OUTPUT_TOKENS": "32000",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1"
  },
  "hasCompletedOnboarding": true
}

OpenAI Codex CLI

Recommended: run the UnityAI wizard and select OpenAI Codex CLI. Codex uses the OpenAI-compatible UnityAI endpoint.

~/.codex/config.toml 
model = "unityai-plus"
model_provider = "unityai"

[model_providers.unityai]
name = "UnityAI"
base_url = "https://api.unityai.cloud/api/v1"
wire_api = "chat"
env_key = "OPENAI_API_KEY"
~/.codex/auth.json 
{
  "OPENAI_API_KEY": "YOUR_API_KEY"
}

OpenCode

Recommended: run the UnityAI wizard and select OpenCode. The wizard writes the provider block in ~/.config/opencode/opencode.json.

~/.config/opencode/opencode.json 
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "unityai": {
      "npm": "@ai-sdk/anthropic",
      "name": "UnityAI",
      "options": {
        "baseURL": "https://api.unityai.cloud/v1",
        "apiKey": "YOUR_API_KEY"
      },
      "models": {
        "unityai-plus": { "name": "UnityAI Plus" },
        "claude-opus-4-6": { "name": "Claude Opus 4.6" },
        "claude-opus-4-7": { "name": "Claude Opus 4.7" },
        "claude-opus-4-8": { "name": "Claude Opus 4.8" },
        "gpt-5.5": { "name": "GPT 5.5" },
        "claude-sonnet-4-6": { "name": "Claude Sonnet 4.6" },
        "gemini-3.1-pro": { "name": "Gemini 3.1 Pro" }
      }
    }
  }
}

Cline Setup

Recommended: run the UnityAI setup wizard and select Cline. If Cline is not installed, install the extension first, then rerun the wizard.

Recommended Wizard Command 
npx -y @unityai/cli@latest
Optional Direct Command 
npx -y @unityai/cli@latest configure --tool cline --api-key YOUR_API_KEY

Manual fallback: use Anthropic-compatible mode with the UnityAI endpoint.

Cline Settings 
{
  "actModeApiProvider": "anthropic",
  "planModeApiProvider": "anthropic",
  "anthropicBaseUrl": "https://api.unityai.cloud/v1",
  "anthropicModelId": "unityai-plus",
  "planModeAnthropicModelId": "unityai-plus"
}

Kilo Code

Recommended: run the UnityAI setup wizard and select Kilo Code. The wizard configures Kilo with the Anthropic-compatible UnityAI endpoint.

Recommended Wizard Command 
npx -y @unityai/cli@latest
Optional Direct Command 
npx -y @unityai/cli@latest configure --tool kilo --api-key YOUR_API_KEY

Manual fallback: add the Anthropic auth block for UnityAI Plus.

Kilo Auth Configuration 
{
  "anthropic": {
    "type": "api-key",
    "apiKey": "YOUR_API_KEY",
    "baseUrl": "https://api.unityai.cloud/v1",
    "model": "unityai-plus"
  }
}

Factory Droid

Recommended: run the UnityAI setup wizard and select Factory Droid. If Droid is missing, install it first, then rerun the wizard.

Recommended Wizard Command 
npx -y @unityai/cli@latest
Optional Direct Command 
npx -y @unityai/cli@latest configure --tool droid --api-key YOUR_API_KEY

Manual fallback: add a custom Anthropic-compatible model entry.

Droid Custom Model 
{
  "model": "unityai-plus",
  "id": "custom:UnityAIPlus-0",
  "baseUrl": "https://api.unityai.cloud/v1",
  "apiKey": "YOUR_API_KEY",
  "displayName": "UnityAI Plus",
  "provider": "anthropic"
}

Manual IDE Setup

Use this section for tools that are not automated by the UnityAI setup wizard, or when you prefer to configure a client manually.

  • VS Code: Use the Claude Code settings file, then restart VS Code.
  • Cursor: Models > Add custom model with Base URL https://api.unityai.cloud/v1 and model ID unityai-plus.
  • Windsurf: AI Provider settings should use https://api.unityai.cloud/v1 as the base URL.
  • Cline: Set apiProvider to anthropic and use the UnityAI base URL with your API key.
  • Roo Code: Set roo-cline.apiProvider to anthropic and use your UnityAI API key.
Cline settings.json snippet 
{
  "cline.apiProvider": "anthropic",
  "cline.anthropicBaseUrl": "https://api.unityai.cloud/v1",
  "cline.apiKey": "YOUR_API_KEY"
}
Roo Code settings.json snippet 
{
  "roo-cline.apiProvider": "anthropic",
  "roo-cline.anthropicBaseUrl": "https://api.unityai.cloud/v1",
  "roo-cline.apiKey": "YOUR_API_KEY"
}
Generic OpenAI-Compatible Client Env 
OPENAI_API_KEY=YOUR_API_KEY
OPENAI_BASE_URL=https://api.unityai.cloud/api/v1
MODEL=unityai-plus

Authentication

Proxy endpoints accept API key authentication through either header. Dashboard endpoints use JWT tokens returned from the login endpoint.

Header Options 
x-api-key: YOUR_API_KEY
# OR
Authorization: Bearer YOUR_API_KEY

Messages API

Create a message with Anthropic-compatible request and response shape. Set stream: true for SSE streaming.

  • Method/Path: POST /v1/messages
  • Auth Required: API Key
Sample JSON Request 
{
  "model": "unityai-plus",
  "max_tokens": 1024,
  "messages": [
    { "role": "user", "content": "Hello, UnityAI!" }
  ],
  "stream": false
}

Token Counting

Count tokens for a message without sending it to a provider.

  • Method/Path: POST /v1/messages/count_tokens
  • Auth Required: API Key
Sample JSON Request 
{
  "model": "unityai-plus",
  "messages": [
    { "role": "user", "content": "How many tokens is this?" }
  ]
}

Dashboard Auth

Use this route to sign in to dashboard sessions.

  • Method/Path: POST /api/auth/login
  • Auth Required: Email + Password
Sample JSON Request 
{
  "email": "YOUR_EMAIL",
  "password": "YOUR_PASSWORD"
}

Build Notes

Built-in search and vision

UnityAI Plus does not require MCP servers or third-party tools for web search and image understanding. UnityAI Plus includes built-in web search and full vision capability, so the model can work with text and image inputs directly.

Troubleshooting

  • •
    Connection errors: Confirm the API server is reachable or verify your client routes properly to https://api.unityai.cloud.
  • •
    Model not found: Use exact model IDs from the Available Models section.
  • •
    Rate limited: Your token window or requests per minute (RPM) cap may be exhausted.
  • •
    Changes not applying: Restart your IDE after any configuration change.
  • •
    Cursor/Windsurf not routing: Make sure the base URL ends with /v1.
